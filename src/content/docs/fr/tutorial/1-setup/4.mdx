Ce tutoriel utilise GitHub pour stocker notre dépôt et se connecter à un hébergeur. Vous pouvez utiliser le fournisseur Git en ligne de votre choix.

:::note
Si vous êtes déjà familiarisé avec git et que vous avez votre propre flux de travail, créez un nouveau dépôt GitHub pour votre projet en utilisant la méthode de votre choix. Passez à la page suivante : ["Déployer votre site sur le web"](/fr/tutorial/1-setup/5/).
:::

## Créer un dépôt sur GitHub

Bien qu'il existe plusieurs façons de stocker votre code local sur GitHub, ce tutoriel vous guidera à travers une méthode qui ne nécessite pas l'utilisation de git en ligne de commande. 

1. Connectez-vous à GitHub.com dans un navigateur et cliquez sur le <kbd>+</kbd> en haut à droite de l'écran pour créer un nouveau dépôt. 

2. Choisissez un nom pour votre dépôt. Ce nom ne doit pas nécessairement être le même que celui de votre dossier de projet.

3. Des options vous sont proposées, mais vous n'avez pas besoin de modifier les valeurs par défaut. Faites défiler vers le bas et cliquez sur le bouton <kbd>Créer un référentiel</kbd>.

4. Plusieurs étapes de configuration vous seront proposées, mais vous n'aurez besoin d'en utiliser aucune. Notez l'URL de votre référentiel. Vous pouvez maintenant quitter cette page sans rien faire.


## Envoyer(commit) votre code local sur GitHub

Dans la dernière section, vous avez modifié le contenu de votre page. Cela signifie que les fichiers de votre projet ont changé, et que VS Code devrait afficher un numéro en haut de l'icône du menu "Source". Cet onglet source est l'endroit où vous irez régulièrement pour mettre à jour vos fichiers sur GitHub.


1. Cliquez sur l'onglet Source Control dans votre VS Code pour voir la liste des fichiers qui ont été modifiés. Si vous voyez un message indiquant que vous devez installer `git`, suivez les instructions fournies, puis rechargez VS Code.

2.  Cliquez sur le menu <kbd>•••</kbd> "3 points" au-dessus du message de validation et choisissez <kbd>Remote</kbd> > <kbd>Add Remote</kbd>.

3. Sélectionner <kbd>Add remote from GitHub</kbd>. Si nécessaire, suivez les étapes d'authentification, puis revenez à VS Code et répétez cette action.

4. Vous devriez voir une liste de tous vos dépôts sur GitHub. Choisissez celui que vous avez créé pour ce projet. Si vous ne voyez pas votre projet, collez directement son URL GitHub. Il peut également vous être demandé de donner un nom local à ce dépôt. Vous pouvez sélectionner le nom de votre choix.

5. En haut du panneau de menu, il y aura un endroit pour entrer un **message de commit** (description des modifications de votre fichier). Tapez sur `initial commit` et appuyez sur le bouton <kbd>Commit</kbd> pour valider ces modifications.

7. **???????** Vous verrez peut-être un message vous indiquant que vous n'avez pas de modifications "staged" commits, et vous demandant si vous voulez les commits. Cliquez sur <kbd>Always</kbd> et continuez.

8. Enfin, la liste des fichiers modifiés devrait être remplacée par un bouton <kbd>Publish</kbd>. Cliquez sur ce bouton pour envoyer vos modifications validées à GitHub.


### Verifier votre projet dans GitHub

Pour vérifier que votre projet est bien stocké sur GitHub, visitez GitHub.com et regardez sous votre compte pour une liste de vos dépôts. Choisissez le nouveau dépôt que vous avez créé et vérifiez qu'il contient les fichiers de votre projet Astro.
