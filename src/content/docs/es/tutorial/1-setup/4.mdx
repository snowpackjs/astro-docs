Este tutorial utilizará GitHub para almacenar nuestro repositorio y conectarse a un host web. Te invitamos a utilizar el proveedor de git en línea de su elección.

:::nota
Si ya estás familiarizado con git y tienes tu propio flujo de trabajo, crea un nuevo repositorio de GitHub para tu proyecto utilizando el método que prefieras. Pasa a la página siguiente: ["Despliegue de su sitio en la Web"](/es/tutorial/1-setup/5/).
:::

## Crear un repositorio en GitHub

Aunque hay algunas maneras de obtener su código local almacenado en GitHub, este tutorial le guiará a través de un método que no requiere el uso de git en la línea de comandos. 

1. Accede a GitHub.com en un navegador y haz clic en el <kbd>+</kbd> de la parte superior derecha de la pantalla para crear un nuevo repositorio. 

2. Elija un nombre para su repositorio. No tiene por qué ser el mismo nombre que la carpeta del proyecto.

3. Se le presentarán opciones, pero no necesita cambiar ninguna de las predeterminadas. Desplácese hacia abajo y pulse el botón <kbd>Create Repository</kbd>.

4. Se le presentarán varios pasos de configuración, pero no necesitará utilizar ninguno de ellos. Anote la URL de su repositorio. Ahora puede salir de esta página sin hacer nada.


## Compila tu código local en GitHub

En la última sección, usted hizo un cambio en el contenido de su página. Esto significa que los archivos de tu proyecto han cambiado, y VS Code debería mostrar un número encima del icono del menú "Fuente". Esta pestaña de fuente es donde irás regularmente para actualizar tus archivos en GitHub.


1. Haga clic en la pestaña Control de código fuente en su VS Code para ver una lista de los archivos que han cambiado. Si aparece un mensaje indicando que necesita instalar `git`, siga las instrucciones que se proporcionan y, a continuación, vuelva a cargar VS Code.

2. Haga clic en el menú <kbd>•••</kbd> "3 puntos" situado encima del mensaje de confirmación y seleccione <kbd>Remote</kbd> > <kbd>Add Remote</kbd>.

3. Seleccione <kbd>Add Remote from GitHub</kbd>. Si es necesario, siga los pasos de autenticación y, a continuación, vuelva a VS Code y repita esta acción.

4. Deberías ver una lista de todos tus repositorios en GitHub. Elige el que creaste para este proyecto. Si no ves tu proyecto, pega directamente su URL de GitHub. Puede que también se te pida que le des un nombre local a este repositorio. Puedes seleccionar el nombre que quieras.

5. En la parte superior del panel de menú, habrá un lugar para introducir un **commit message** (descripción de los cambios de su archivo). Escriba `initial commit` y pulse el botón <kbd>Commit</kbd> para confirmar los cambios.

7. Puede que aparezca un mensaje diciéndote que no tienes "staged" commits, y preguntándote si quieres escanearlos. Haz clic en<kbd>Always</kbd> y continúa.

8. Por último, la lista de archivos modificados debería sustituirse por un botón <kbd>Publish</kbd>. Haz click en él para enviar los cambios confirmados a GitHub.


### Vea su proyecto en GitHub

Para verificar que tu proyecto está almacenado correctamente en GitHub, visita GitHub.com y busca en tu cuenta una lista de tus repositorios. Elige el nuevo que has creado y comprueba que contiene los archivos de tu proyecto Astro.
