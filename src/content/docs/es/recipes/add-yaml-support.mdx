Astro se basa en Vite y admite complementos de Vite y Rollup. Esta receta utiliza un complemento Rollup para agregar la capacidad de importar un archivo YAML (`.yml`) en Astro.

## Recipe

1. Instalar `@rollup/plugin-yaml`:

    ```bash
    npm install @rollup/plugin-yaml --save-dev
    ```

2. Importe el complemento en su `astro.config.mjs` y agréguelo a la array de complementos de Vite:

    ```js title="astro.config.mjs" ins={2,5-7}
    import { defineConfig } from 'astro/config';
    import yaml from '@rollup/plugin-yaml';

    export default defineConfig({
      vite: {
        plugins: [yaml()]
      }
    });
    ```


3. Finalmente, puedes importar datos YAML usando una declaración `import`:

    ```js
    import yml from './data.yml';
    ```

    :::note
    Si bien ahora puedes importar datos YAML en tu proyecto Astro, tu editor no proporcionará tipos para los datos importados. Para agregar tipos, cree o busque un archivo `*.d.ts` existente en el directorio `src` de su proyecto y agregue lo siguiente:
    ```ts title="src/files.d.ts"
    // Especifique la extensión del archivo que desea importar
    declare module "*.yml" {
      const value: any; // Agregue definiciones de tipo aquí si lo desea
      export default value;
    }
    ```
    Esto permitirá que su editor proporcione sugerencias de tipo para sus datos YAML.
    :::
