[Front Matter CMS](https://frontmatter.codes/)将CMS集成到您的编辑器中，它可以在Visual Studio Code、GitPod和许多其他编辑器中运行。

## 与Astro集成

在本节中，我们将介绍如何将Front Matter CMS添加到您的Astro项目中。

### 前提条件 

- Visual Studio Code
- 使用[Astro博客模板](https://github.com/withastro/astro/tree/main/examples/blog)来提供基本配置和示例内容，以便开始使用Front Matter CMS。

### 安装Front Matter CMS扩展

您可以从[Visual Studio Code Marketplace - Front Matter](https://marketplace.visualstudio.com/items?itemName=eliostruyf.vscode-front-matter)获取扩展，或者点击以下链接：[在VS Code中打开Front Matter CMS扩展](vscode:extension/eliostruyf.vscode-front-matter)

### 项目初始化

安装Front Matter CMS后，您将在Activity Bar中获得一个新的图标。单击该图标将在主侧边栏中打开**Front Matter CMS**面板。按照下面的步骤来初始化您的项目：

- 在Front Matter面板上点击**Initialize project**按钮
- 欢迎屏幕将打开，您可以开始初始化项目
- 点击第一步**Initialize project**
- 由于Astro是受支持的框架之一，您可以从列表中选择它
- 注册您的内容文件夹，在本例中是`src/content/blog`文件夹。

:::note
文件夹注册是必需的，以便让Front Matter CMS知道在哪里找到和创建您的内容。您可以有多种类型的文件夹，比如页面、博客、文档等等。
:::

- 您将被要求输入文件夹的名称。默认情况下，它会使用文件夹名称。

:::note
名称将在新内容的创建过程中使用。例如，有多个文件夹注册可以让您选择要创建的内容类型。
:::

- 点击**Show the dashboard**以打开内容仪表板

:::tip
一旦Front Matter CMS初始化完成，您可以按照以下方式打开仪表板：

- 使用键盘绑定：<kbd>alt</kbd> + <kbd>d</kbd>（Windows和Linux）或<kbd>options</kbd> + <kbd>d</kbd>（macOS）
- 打开命令面板，搜索`Front Matter: Open dashboard`
- 单击面板标题栏或文件上的**Front Matter**图标。
:::

### 项目配置

项目初始化后，您将在项目的根目录中获得一个`frontmatter.json`配置文件和一个`.frontmatter`文件夹。

#### 内容类型配置

内容类型是Front Matter CMS管理您的内容的方式。每个内容类型包含一组字段，可以根据您希望在网站上使用的每种类型的内容来定义。

这些字段对应于页面内容的前置信息。

您可以在`frontmatter.json`文件中配置内容类型。

- 打开`frontmatter.json`文件
- 用以下内容类型配置替换`frontMatter.taxonomy.contentTypes`数组：

```json title="frontmatter.json"
"frontMatter.taxonomy.contentTypes": [
  {
    "name": "default",
    "pageBundle": false,
    "previewPath": "'blog'",
    "filePrefix": null,
    "fields": [
      {
        "title": "Title",
        "name": "title",
        "type": "string",
        "single": true
      },
      {
        "title": "Description",
        "name": "description",
        "type": "string"
      },
      {
        "title": "Publishing date",
        "name": "pubDate",
        "type": "datetime",
        "default": "{{now}}",
        "isPublishDate": true
      },
      {
        "title": "Content preview",
        "name": "heroImage",
        "type": "image",
        "isPreviewImage": true
      }
    ]
  }
]
```

:::note
此配置确保 Front Matter 内容类型与 Astro 博客模板的内容集合模式匹配。
:::

:::tip
您可以在 Front Matter CMS 的内容创建文档部分中找到有关内容类型和支持的字段的更多信息。
:::

### 在编辑器中预览文章

从 **Front Matter CMS** 面板，点击 **Start server** 按钮。此操作将启动 Astro 本地开发服务器。一旦运行，您可以打开内容仪表板，选择一篇文章，然后点击 **Open preview** 按钮将文章在编辑器中打开。

### 创建新文章

打开 **Front Matter CMS Dashboard**，您可以按照以下方式操作： 

- 打开 `Front Matter CMS` 的内容仪表板
- 点击 **Create content** 按钮
- Front Matter 将要求您输入文章的标题。输入标题并按 Enter 键
- 您的新文章将被创建并在编辑器中打开。您可以开始编写文章

### 使用 Markdoc 与 Front Matter CMS

要在 Front Matter CMS 中使用 Markdoc，您必须在 `frontMatter.content.supportedFileTypes` 中进行配置。此设置使 CMS 知道它可以处理哪种类型的文件。

您可以按以下方式配置此设置：

```json title="frontmatter.json"
"frontMatter.content.supportedFileTypes": [ "md", "markdown", "mdx", "mdoc" ]
```

要允许将内容创建为 Markdoc，请在内容类型上指定 `fileType` 属性。

```json title="frontmatter.json" ins={7}
"frontMatter.taxonomy.contentTypes": [
  {
    "name": "default",
    "pageBundle": false,
    "previewPath": "'blog'",
    "filePrefix": null,
    "fileType": "mdoc",
    "fields": [
      {
        "title": "Title",
        "name": "title",
        "type": "string",
        "single": true
      },
      {
        "title": "Description",
        "name": "description",
        "type": "string"
      },
      {
        "title": "Publishing date",
        "name": "pubDate",
        "type": "datetime",
        "default": "{{now}}",
        "isPublishDate": true
      },
      {
        "title": "Content preview",
        "name": "heroImage",
        "type": "image",
        "isPreviewImage": true
      }
    ]
  }
]
```
