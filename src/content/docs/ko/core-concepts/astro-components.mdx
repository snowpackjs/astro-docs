**아스트로 컴포넌트**는 모든 아스트로 프로젝트의 기본 구성 요소입니다. 아스트로 컴포넌트는 클라이언트 측 런타임이 없는 HTML 전용 템플릿 컴포넌트입니다. Astro 컴포넌트는 파일 확장자 '.astro'로 식별할 수 있습니다.

아스트로 컴포넌트는 매우 유연합니다. 종종 Astro 컴포넌트에는 헤더나 프로필 카드와 같이 페이지에서 **재사용 가능한 UI**가 포함되어 있을 수도 있고 때로는 Astro 컴포넌트에 SEO 작업을 쉽게 해주는 일반적인 `<meta>` 태그 모음과 같은 작은 HTML 스니펫이 포함될 수도 있습니다. Astro 컴포넌트는 전체 페이지 레이아웃을 포함할 수도 있습니다.

Astro 컴포넌트에 대해 알아야 할 가장 중요한 점은 Astro 컴포넌트들은 **클라이언트에서 렌더링되지 않는다는 것입니다**. 빌드 시, 또는 필요할 때(on demand) [server-side rendering (SSR)](/ko/guides/server-side-rendering/)을 사용하여 HTML로 렌더링합니다. 또한 컴포넌트 프론트메터에 JavaScript 코드를 포함할 수 있으며, 사용자의 브라우저로 전송되는 최종 페이지에서는 모든 코드가 제거됩니다. 그 결과, 기본적으로 JavaScript의 발자취가 없는 더 빠른 사이트를 만들 수 있습니다.

Astro 컴포넌트에 클라이언트 측 인터랙티브 기능이 필요한 경우 [standard HTML `<script>` tags](/ko/guides/client-side-scripts/) 또는 [UI Framework components](/ko/core-concepts/framework-components/#hydrating-interactive-components)를 추가하면 됩니다.


## 컴포넌트 구조

아스트로 컴포넌트는 **컴포넌트 스크립트**와 **컴포넌트 템플릿**의 두 가지 주요 부분으로 구성됩니다. 각 파트는 서로 다른 작업을 수행하지만, 함께 사용하면 어떤 빌드든 처리할 수 있을 만큼 사용하기 쉽고 표현력이 풍부한 프레임워크를 제공합니다.

```astro title="src/components/EmptyComponent.astro"
---
// Component Script (JavaScript)
---
<!-- Component Template (HTML + JS Expressions) -->
```

### 컴포넌트 스크립트

Astro는 코드 펜스(`---`)를 사용하여 Astro 컴포넌트에서 컴포넌트 스크립트를 식별합니다. 마크다운을 작성해 본 적이 있다면 *프론트매터*라는 유사한 개념에 이미 익숙할 것입니다. Astro의 컴포넌트 스크립트 아이디어는 이 개념에서 직접 영감을 얻었습니다.

컴포넌트 스크립트를 사용하여 템플릿을 렌더링하는 데 필요한 JavaScript 코드를 작성할 수 있습니다. 이는 다음이 포함될 수 있습니다.

- 다른 아스트로 컴포넌트들 가져오기
- React 같은 다른 프레임워크 컴포넌트들 가져오기
- JSON 파일 같은 데이터 파일 가져오기
- API나 데이터베이스에서 컨텐츠 가져오기
- 템플릿에서 참조할 변수 만들기


```astro title="src/components/MyComponent.astro"
---
import SomeAstroComponent from '../components/SomeAstroComponent.astro';
import SomeReactComponent from '../components/SomeReactComponent.jsx';
import someData from '../data/pokemon.json';

// <X title="Hello, World">와 같이 컴포넌트에 전달된 프롭에 접근합니다
const {title} = Astro.props;
// 외부로부터 데이터를 가져옵니다, 프라이빗 API나 데이터베이스에서 부터도 가져올 수 있어요! 
const data = await fetch('SOME_SECRET_API_URL/users').then(r => r.json());
---
<!-- 당신이 작성한 템플릿 -->
```

코드 펜스는 작성하는 JavaScript가 "펜스 인"되도록 보장하도록 설계되었습니다. 즉, 이는 프론트엔드 애플리케이션으로 빠져나가거나 사용자의 손에 넘어가지 않는다는 것을 의미합니다. 펜스 안에서는 비용이 많이 들거나 민감한 코드(비공개 데이터베이스에 대한 호출 등)를 안전하게 작성할 수 있으며, 사용자의 브라우저에 노출될 염려도 없습니다.

:::tip
컴포넌트 스크립트에서 TypeScript를 작성할 수도 있습니다!
:::

### 컴포넌트 템플릿

컴포넌트 템플릿은 코드 펜스 아래에 있으며 컴포넌트의 HTML 출력을 결정합니다.

여기에 일반 HTML을 작성하면 컴포넌트는 그 HTML을 가져와서 그것을 사용하는 모든 Astro 페이지에서 해당 HTML을 렌더링합니다.

그리고 [Astro's component template syntax](/ko/core-concepts/astro-syntax/)는 또한 **JavaScript 표현식**, 아스트로 [`<style>`](/ko/guides/styling/#styling-in-astro) 태그, [`<script>`](/ko/guides/client-side-scripts/#using-script-in-astro) 태그, **임포트된 컴포넌트** 및 [**special Astro directives**](/ko/reference/directives-reference/)도 지원합니다. 컴포넌트 스크립트에 정의된 데이터와 값은 컴포넌트 템플릿에서 동적으로 생성된 HTML을 생성하는 데 사용할 수 있습니다.

```astro title="src/components/MyFavoritePokemon.astro"
---
// 컴포넌트 스크립트들!
import Banner from '../components/Banner.astro';
import ReactPokemonComponent from '../components/ReactPokemonComponent.jsx';
const myFavoritePokemon = [/* ... */];
const { title } = Astro.props;
---
<!-- HTML 주석 지원됨 -->
{/* JS 주석 구문도 또한 지원됨! */}

<Banner />
<h1>Hello, world!</h1>

<!-- 컴포넌트 스크립트에서 프롭 및 기타 변수를 사용함 -->
<p>{title}</p>

<!-- `client:` 지시어를 사용하여 다른 UI 프레임워크 컴포넌트를 포함시켜 하이드레이트함 -->
<ReactPokemonComponent client:visible />

<!-- JSX와 유사하게 HTML과 자바스크립트 표현식을 혼합 -->
<ul>
  {myFavoritePokemon.map((data) => <li>{data.name}</li>)}
</ul>

<!-- 템플릿 지시문을 사용하여 여러 문자열 또는 객체에서 클래스 이름을 빌드하는 것도 가능! -->
<p class:list={["add", "dynamic", {classNames: true}]} />
```

## 컴포넌트 기반 설계

컴포넌트는 **재사용 가능**하고 **페이지를 구성**할 수 있도록 설계되었습니다. 다른 컴포넌트 내부에 컴포넌트를 사용하여 점점 더 고급 UI를 구축할 수 있습니다. 예를 들어, `Button` 컴포넌트를 사용해 `ButtonGroup` 컴포넌트를 만들 수 있습니다.

```astro title="src/components/ButtonGroup.astro"
---
import Button from './Button.astro';
---
<div>
  <Button title="Button 1" />
  <Button title="Button 2" />
  <Button title="Button 3" />
</div>
```


## 컴포넌트 프롭

Astro 컴포넌트는 프롭을 정의하고 받을 수 있습니다. 덕분에 컴포넌트 템플릿에서는 이러한 프롭들을 사용하여 HTML을 렌더링할 수 있습니다. 프롭은 프론트메터 스크립트의 `Astro.props` 글로벌에서 사용할 수 있습니다.

다음은 `greeting` 프롭과 `name` 프롭을 수신하는 컴포넌트의 예시입니다. 수신할 프롭은 글로벌 `Astro.props` 객체에서 디스트럭처링된다는 점에 유의하세요.

```astro "Astro.props"
---
// src/components/GreetingHeadline.astro
// 사용법: <GreetingHeadline greeting="Howdy" name="Partner" />
const { greeting, name } = Astro.props;
---
<h2>{greeting}, {name}!</h2>
```

이 컴포넌트를 가져와서 다른 Astro 컴포넌트, 레이아웃 또는 페이지에서 렌더링할 때 이러한 프롭을 속성으로 전달할 수 있습니다.

```astro /(\w+)=\S+/
---
// src/components/GreetingCard.astro
import GreetingHeadline from './GreetingHeadline.astro';
const name = "Astro"
---
<h1>Greeting Card</h1>
<GreetingHeadline greeting="Hi" name={name} />
<p>I hope you have a wonderful day!</p>
```

또한 `Props` 타입 인터페이스를 사용하여 타입스크립트로 프롭을 정의할 수도 있습니다. Astro는 프론트매터에서 `Props` 인터페이스를 자동으로 선택해 타입 경고/오류를 제공합니다. 이 프롭들은 `Astro.props`에서 디스트럭처링할 때 기본값을 지정할 수도 있습니다.

```astro ins={3-6}
---
// src/components/GreetingHeadline.astro
interface Props {
  name: string;
  greeting?: string;
}

const { greeting = "Hello", name } = Astro.props;
---
<h2>{greeting}, {name}!</h2>
```

컴포넌트 프롭에 기본값을 지정하여 프롭이 제공되지 않을 때 사용할 수 있습니다.

```astro ins="= \"Hello\"" ins="= \"Astronaut\""
---
// src/components/GreetingHeadline.astro
const { greeting = "Hello", name = "Astronaut" } = Astro.props;
---
<h2>{greeting}, {name}!</h2>
```

## Slots

`<slot />` 요소는 외부 HTML 콘텐츠의 플레이스홀더로, 다른 파일의 하위 요소를 컴포넌트 템플릿에 삽입(슬롯)할 수 있습니다.

기본적으로 컴포넌트에 전달된 모든 하위 요소는 `<slot />`에 렌더링됩니다.

:::note
Astro 컴포넌트에 전달되는 어트리뷰트인 _props_ 와 달리, _slots_ 는 `Astro.props`와 함께 컴포넌트 전체에서 사용할 수 있는 자식 HTML 요소를 작성된 위치에 렌더링합니다.
:::

```astro "<slot />"
---
// src/components/Wrapper.astro
import Header from './Header.astro';
import Logo from './Logo.astro';
import Footer from './Footer.astro';

const { title } = Astro.props
---
<div id="content-wrapper">
  <Header />
  <Logo />
  <h1>{title}</h1>
  <slot />  <!-- 하위 요소들은 여기로 갈 것입니다 -->
  <Footer />
</div>
```

```astro {6-7}
---
// src/pages/fred.astro
import Wrapper from '../components/Wrapper.astro';
---
<Wrapper title="Fred's Page">
  <h2>All about Fred</h2>
  <p>Here is some stuff about Fred.</p>
</Wrapper>
```

이 패턴은 [Astro layout component](/ko/core-concepts/layouts/)의 기본입니다. 전체 HTML 콘텐츠 페이지를 `<SomeLayoutComponent></SomeLayoutComponent>` 태그로 "래핑"하고 이를 컴포넌트로 전송하여 페이지 안의 요소들을 렌더할 수 있습니다.


### 이름이 있는 Slots

Astro 컴포넌트에는 이름이 있는 슬롯을 가질 수도 있습니다. 이렇게 하면 해당 슬롯 이름을 가진 HTML 요소만 슬롯의 위치로 전달할 수 있습니다.

슬롯은 아래의 예와 같이 `name` 속성을 사용하여 이름이 지정됩니다.

```astro /<slot .*?/>/
---
// src/components/Wrapper.astro
import Header from './Header.astro';
import Logo from './Logo.astro';
import Footer from './Footer.astro';

const { title } = Astro.props
---
<div id="content-wrapper">
  <Header />
  <slot name="after-header"/>  <!--  `slot="after-header"` 속성을 가진 하위 요소가 여기에 올 것입니다 -->
  <Logo />
  <h1>{title}</h1>
  <slot />  <!--  `slot` 속성이 없는 하위 요소나 `slot="default"` 속성을 가진 하위 요소가 여기에 올 것입니다 -->
  <Footer />
  <slot name="after-footer"/>  <!--  `slot="after-footer"` 속성을 가진 하위 요소가 여기에 올 것입니다 -->
</div>
```

특정 슬롯에 HTML 콘텐츠를 삽입하려면 하위 요소에 `slot` 속성을 사용하여 슬롯의 이름을 지정합니다. 컴포넌트의 다른 모든 자식 요소는 기본(이름 없는) `<slot />`에 삽입됩니다.


```astro /slot=".*?"/
---
// src/pages/fred.astro
import Wrapper from '../components/Wrapper.astro';
---
<Wrapper title="Fred's Page">
  <img src="https://my.photo/fred.jpg" slot="after-header">
  <h2>All about Fred</h2>
  <p>Here is some stuff about Fred.</p>
  <p slot="after-footer">Copyright 2022</p>
</Wrapper>
```

컴포넌트에서 일치하는 `<slot name="my-slot" />` 플레이스홀더를 통해 전달하려는 하위 요소에 `slot="my-slot"` 속성을 사용하세요.

:::tip
이름있는 슬롯은 [UI framework components](/ko/core-concepts/framework-components/)에도 전달할 수 있습니다!
:::


### 슬롯의 대체 콘텐츠
슬롯은 **대체 콘텐츠**를 렌더링할 수도 있습니다. 슬롯에 전달된 일치하는 자식이 없는 경우 `<slot />` 요소는 자체 플레이스홀더 자식을 렌더링합니다.

```astro {14}
---
// src/components/Wrapper.astro
import Header from './Header.astro';
import Logo from './Logo.astro';
import Footer from './Footer.astro';

const { title } = Astro.props
---
<div id="content-wrapper">
  <Header />
  <Logo />
  <h1>{title}</h1>
  <slot>
    <p>This is my fallback content, if there is no child passed into slot</p>
    <!-- 만약 슬롯에 아무것도 전달되지 않았다면 이것이 표시될 것입니다 -->
  </slot>
  <Footer />
</div>
```


## HTML 컴포넌트

Astro는 `.html` 파일을 컴포넌트로 가져와서 사용하거나 이러한 파일을 `src/pages/` 하위 디렉토리에 페이지로 배치하는 기능을 지원합니다. 프레임워크 없이 구축된 기존 사이트의 코드를 재사용하거나 컴포넌트에 동적 기능이 없는지 확인하려는 경우 HTML 컴포넌트를 사용할 수 있습니다.

HTML 컴포넌트에는 유효한 HTML만 포함되어야 하며, 따라서 다음과 같은 Astro 컴포넌트의 주요 기능이 없습니다.
- 프론트매터, 서버 측 가져오기 또는 동적 표현식을 지원하지 않습니다.
- 모든 `<script>` 태그는 번들되지 않은 상태로 유지되며, `is:inline`이 있는 것처럼 취급됩니다. 
- 이 컴포넌트는 [`public/` 폴더에 있는 참조 자산](/ko/guides/images/#public)만 참조할 수 있습니다.

:::note
HTML 컴포넌트 내부의 [`<slot />` 요소](/en/core-concepts/astro-components/#slots)는 Astro 컴포넌트에서와 마찬가지로 작동합니다. 대신 [HTML 웹 컴포넌트 슬롯](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/slot) 요소를 사용하려면 `<slot>` 요소에 `is:inline`을 추가하세요.
:::
